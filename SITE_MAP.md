# サイト構成・コンテンツマップ

マクセラス (maxelustech.com) コーポレートサイトの **ページツリーと各ページの実表示テキスト** をまとめたドキュメント。文言を修正したいときの起点として使う。

- 表示文言の実データは `app/data/*.ts` と各ページの `page.tsx` にハードコードされている (CMS は休止中)。
- このドキュメントは「訪問者が画面で目にするテキスト」を中心に整理している。実装の詳細・規約は [CLAUDE.md](./CLAUDE.md) を参照。

課題解決型の営業 LP 構成。上から 8 セクション。各セクションには背景に薄い英字 (HOME / APPROACH / SERVICES …) が入る。

**Solution の 3 カード** (01/02/03)
- 完全オーダーメイド — テンプレートではなく、お客様のビジネス課題に合わせた完全カスタム設計
- 企画から運用まで伴走 — リリースして終わりではなく、運用・改善まで継続的にサポート
- 成果から逆算 — KPIを明確にし、ビジネス成果につながる機能を優先して開発

**Services の 5 カード** (01〜05、リンク先)
- 完全オーダーメイドアプリ開発 — AI × 最新手法。企画から運用まで伴走 → `/lp/full-order-app-development`
- 業務DX設計・開発 — 業務の「型」を見つけて、システム化の道筋を → `/lp/business-dx-design`
- AI機能組込み・自動化 — 要約/分類/検索/生成/動画分析。AIで業務を自動化 → `/services`
- ホームページ制作 — 普通のHPも、3D/アニメーションみたいな『尖った表現』も対応 → `/services`
- プロダクト開発 — 図面コネクト、インタラクティブWeb3Dなど → `/services`

**Challenge の Before → After**
- Before: 仕様が固まらない / 途中で手戻りが出る / 追加費用が怖い
- After: 触れるデモで合意 / 設計確定→開発 / 予算と範囲が明確

**Process の 6 ステップ**
1. 課題 —「システム開発を依頼したいけど、完成イメージが分からない」「途中で認識のズレが発生し、手戻りが発生」「予算オーバーで困った経験がある」
2. 無料相談 — まずは課題を整理するところから。あなたのビジネスの課題をヒアリングし、システム化すべきポイントを一緒に見つけていきます。相談は完全無料です。
3. デモ提示 — 打合せ後、すぐに画面デモを作成。実際に触りながら認識を合わせるから、ズレが起きにくい。見積りも明確で安心です。
4. 発注 — デモで合意した内容をベースに、正式発注。見積りも明確で、追加費用の心配もありません。
5. 開発 — 詳細設計を確定し、開発を進めます。進捗は随時共有し、透明性のあるプロセスで進行。
6. 納品・運用 — 納品後も半年間は並走サポート。運用定着まで一緒に伴走します。「作って終わり」ではありません。
   - 最終ステップの CTA: 「無料相談する」「サービスを見る」

**ServiceTypes の 3 カード**
- ホームページ — WordPressではできない体験型Web → `/services`
- Webアプリ — 業務効率化・DXアプリ (「デモを表示」でサロン予約アプリのモーダル)
- iOSアプリ — ネイティブアプリ開発 → `/lp/full-order-app-development`

**CTA セクション本文**: オーダーメイドのアプリとホームページで、競合に差をつけ、成長を加速させましょう。まずは無料相談で、課題整理→触れるデモ提示まで一緒に進めます。
